= How to create Podman secrets
:toc: left
:toclevels: 2
:sectanchors:
:sectlinks:
:icons: font

Six databases in this collection use `Secret=` directives instead of inline environment variables.
Podman secrets are stored encrypted at rest and never appear in `podman inspect` output.

The general form for all of them:

[source,bash]
----
echo -n "<value>" | podman secret create <secret-name> -
----

Verify after creation:

[source,bash]
----
podman secret ls
----

== Secrets by database

=== Neo4j — `neo4j-auth`

Format: `neo4j/<password>` to set the password for the built-in `neo4j` user.
Use the literal string `none` to disable authentication (development only).

[source,bash]
----
echo -n "neo4j/changeme" | podman secret create neo4j-auth -
----

=== Apache AGE — `apache-age-password`

Standard PostgreSQL password for the `age_user` account.

[source,bash]
----
echo -n "changeme" | podman secret create apache-age-password -
----

=== AllegroGraph — `allegro-graph-password`

Password for the `admin` superuser.

[source,bash]
----
echo -n "changeme" | podman secret create allegro-graph-password -
----

=== Apache Jena Fuseki — `jena-fuseki-admin-password`

Password for the Fuseki admin interface.

[source,bash]
----
echo -n "changeme" | podman secret create jena-fuseki-admin-password -
----

=== ArcadeDB — `arcade-db-root-password`

Root server password.

NOTE: The environment variable target is `arcadedb.server.rootPassword` (dots in the name), which ArcadeDB maps directly from environment variable names using dot notation.

[source,bash]
----
echo -n "changeme" | podman secret create arcade-db-root-password -
----

=== RDFox — `rdfox-license`

RDFox requires a base64-encoded license issued by Oxford Semantic Technologies.

[source,bash]
----
ENCODED=$(base64 -w0 /path/to/rdfox.lic)
echo -n "$ENCODED" | podman secret create rdfox-license -
----

== Rotate a secret

Podman does not support in-place secret updates.
Delete and recreate:

[source,bash]
----
podman secret rm <secret-name>
echo -n "<new-value>" | podman secret create <secret-name> -
systemctl --user restart <database>.service
----

== Read a secret (for verification)

[source,bash]
----
podman secret inspect <secret-name> \
  --format "{{.Name}}: created {{.CreatedAt}}"
----

WARNING: Podman does not expose secret values after creation. If you need to check the actual value, re-create from your source.

== Store secrets from a file

For secrets you manage outside Podman (e.g. with a password manager or Vault):

[source,bash]
----
podman secret create neo4j-auth - < /run/secrets/neo4j-auth.txt
----

Restrict permissions on source files:

[source,bash]
----
chmod 600 /run/secrets/neo4j-auth.txt
----
