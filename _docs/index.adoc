Podman quadlet definitions for 23 graph database engines, structured for systemd-managed deployment on a Linux desktop or server.

== What's here

Each database engine lives in its own subdirectory and ships three kinds of files:

[cols="<2,<5",options="header"]
|===
| File type | Purpose

| `<name>.container`
| Systemd quadlet: image, ports, volumes, secrets, health check

| `<name>-data.volume`
| Systemd quadlet: declares a named Podman volume

| `.dotter/global.toml` / `.dotter/local.toml`
| Dotter configuration: maps source files to symlink targets in `~/.config/containers/systemd/pgdb/`
|===

The `aerospike-graph/` directory also contains a `.network` file and a second `.container` for the backend Aerospike Server process.
