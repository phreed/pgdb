== apache-age

PostgreSQL with the Apache AGE graph extension baked in, enabling openCypher graph queries alongside standard SQL.
After connecting, load the extension and set the search path before issuing graph queries:

[source,sql]
----
CREATE EXTENSION age;
LOAD 'age';
SET search_path = ag_catalog, "$user", public;
----

See https://age.apache.org/getstarted/quickstart[age.apache.org quickstart].

[cols="<2,<5",frame=none,grid=rows]
|===
| Directory | `apache-age/`
| Image | `docker.io/apache/age:latest`
| Port | 5455 (PostgreSQL; native 5432)
| Volume | `apache-age-data` → `/var/lib/postgresql/data`
| Environment | `POSTGRES_USER=age_user`, `POSTGRES_DB=age_db`
| Secrets | `apache-age-password` → `POSTGRES_PASSWORD`
| Protocol | PostgreSQL wire protocol, SQL + openCypher via AGE
| Health check | `pg_isready -U age_user -d age_db`
|===

== pggraph

PostgreSQL with graph traversal built in.
Provides bounded traversal, shortest-path, and relationship queries directly on ordinary PostgreSQL tables using compressed-sparse-row (CSR) indexes as derived, rebuildable state.
No separate graph storage — PostgreSQL remains the authoritative source.
See https://github.com/Evokoa/pgGraph[github.com/Evokoa/pgGraph].

[cols="<2,<5",frame=none,grid=rows]
|===
| Directory | `pggraph/`
| Image | `ghcr.io/evokoa/pggraph:0.1.7`
| Port | 5432 (PostgreSQL)
| Volume | `pggraph-data` → `/var/lib/postgresql/data`
| Environment | `POSTGRES_USER=postgres`, `POSTGRES_DB=graph`
| Secrets | `pggraph-password` → `POSTGRES_PASSWORD`
| Protocol | PostgreSQL wire protocol; graph functions via SQL
| Health check | `pg_isready -U postgres -d graph`
| Status | Early alpha
|===
