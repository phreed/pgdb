Six databases use `Secret=` directives.
This page lists every secret name, the environment variable it maps to, the expected format, and the creation command.

See xref:../how-to/create-secrets.adoc[Create Podman secrets] for step-by-step instructions.

== Creation commands

[source,bash]
----
# Neo4j
echo -n "neo4j/changeme" | podman secret create neo4j-auth -

# Apache AGE
echo -n "changeme" | podman secret create apache-age-password -

# AllegroGraph
echo -n "changeme" | podman secret create allegro-graph-password -

# Jena Fuseki
echo -n "changeme" | podman secret create jena-fuseki-admin-password -

# ArcadeDB
echo -n "changeme" | podman secret create arcade-db-root-password -

# RDFox (base64-encode the license file first)
base64 -w0 /path/to/rdfox.lic | podman secret create rdfox-license -
----

== Secret lifecycle

`podman secret ls`::
  List all secrets.

`podman secret inspect <secret-name>`::
  Show metadata only — values are not retrievable after creation.

`podman secret rm <secret-name>`::
  Delete a secret.

.Rotate a secret (delete and recreate)
[source,bash]
----
podman secret rm neo4j-auth
echo -n "neo4j/newpassword" | podman secret create neo4j-auth -
systemctl --user restart neo4j.service
----

== Notes

`arcade-db-root-password`::
  The target variable name `arcadedb.server.rootPassword` contains dots.
  This is intentional — ArcadeDB reads its configuration directly from environment variable names using dot notation, and Podman passes the name through unchanged.

`rdfox-license`::
  The license file must be base64-encoded with no line breaks (`-w0` flag).
  RDFox decodes the environment variable at startup.
  If the container fails to start with exit code 1, verify the encoding:
+
[source,bash]
----
echo "$RDFOX_LICENSE_CONTENT" | base64 -d | head -c 20
----
+
This should show recognisable license file content.

`neo4j-auth`::
  The `none` value disables all authentication.
  Use this only in isolated development environments.

WARNING: When switching Neo4j from a password to `none` (or vice versa), delete the existing data volume first — Neo4j embeds auth configuration in the store.
